# 水果消消乐

一个基于 React + Vite 实现的单页 Match 3 三消小游戏。玩家通过交换相邻水果，让横向或纵向 3 个及以上同类水果连成一线并消除，在限定步数内冲击目标分数。

## 项目特点

- 经典三消玩法：点击两个相邻水果进行交换，形成三连或更多即可消除。
- 8 x 8 棋盘：使用二维数组维护棋盘状态，渲染为 CSS Grid。
- 自动生成可玩的初始棋盘：开局生成时会避免直接出现三连。
- 连锁消除计分：水果下落后如果继续形成匹配，会按连锁倍率获得更高分数。
- 自动提示：开局会短暂高亮一组可交换水果。
- 无解刷新：当棋盘没有任何可用交换时，会自动刷新棋盘。
- 游戏说明弹窗：支持点击帮助按钮查看规则，支持 Escape 关闭。
- 响应式布局：针对移动端做了基础适配。

## 技术栈

- React 18
- React DOM 18
- Vite 6
- JavaScript ES Module
- 普通 CSS

当前项目没有后端服务，游戏逻辑全部在浏览器端运行。

## 游戏规则

- 点击一个水果后，再点击与它上下左右相邻的另一个水果即可尝试交换。
- 只有交换后形成横向或纵向 3 个及以上同类水果时，本次交换才算有效。
- 每次有效交换会消耗 1 步。
- 每消除 1 个水果获得 40 分。
- 如果掉落后的新棋盘继续形成匹配，会触发连锁消除。
- 连锁得分公式为：`消除数量 x 40 x 连锁轮次`。
- 在 26 步内达到 1800 分即通关成功。
- 步数用完且没有达到目标分数则游戏失败。

## 核心逻辑说明

### 棋盘生成

`createBoard()` 会生成一个 8 x 8 的二维数组。生成每个格子时，会检查左侧和上方是否已经连续出现两个相同水果，从而避免开局直接出现三连。

### 匹配检测

`findMatches(board)` 会分别扫描横向和纵向连续水果：

- 横向从每一行左到右扫描。
- 纵向从每一列上到下扫描。
- 连续 3 个及以上相同水果会被记录到 `Set` 中。
- 每个匹配格子的 key 格式为 `"row,col"`。

### 交换与校验

玩家点击两个格子后：

1. `areNeighbors(first, second)` 判断两个格子是否相邻。
2. `swapCells(board, first, second)` 生成交换后的新棋盘。
3. `findMatches(swappedBoard)` 判断交换后是否产生消除。
4. 如果没有产生消除，本次交换无效，不扣步数。
5. 如果产生消除，扣除 1 步并进入结算流程。

### 消除与下落

`resolveBoard()` 负责处理一次有效交换后的完整结算：

1. 标记需要消除的格子并播放动画。
2. `removeMatches()` 将匹配格子置为 `null`。
3. `dropCandies()` 让上方水果向下掉落，并在顶部补充新水果。
4. 再次调用 `findMatches()` 检查是否产生连锁。
5. 重复以上过程，直到棋盘没有新的匹配。

### 胜负判断

`checkGameState()` 会在每轮结算后检查：

- 分数达到目标：通关成功。
- 步数用完：游戏失败。
- 棋盘无可用交换：自动刷新棋盘。
- 否则继续游戏。

## 样式与交互

- 棋盘使用 CSS Grid 实现。
- 水果格子是按钮元素，便于键盘焦点和无障碍语义扩展。
- 消除动画使用 `pop` keyframes。
- 提示动画使用 `hint` keyframes。
- 帮助弹窗使用 `role="dialog"` 和 `aria-modal="true"`。
- 状态消息使用 `aria-live="polite"`，方便屏幕阅读器感知分数和操作反馈。

## 新增水果的步骤

如果要新增水果类型，需要同步修改以下位置：

1. 将新的图片资源放入 `src/assets/fruits/`。
2. 在 `src/App.jsx` 顶部添加图片 import。
3. 向 `fruits` 数组追加 `{ name, image }`。
4. 更新 `candyTypes` 的数量。

注意：`fruits` 数组下标会作为棋盘里的水果类型值，所以数组顺序和游戏状态有关。

## 开发建议

- 修改游戏逻辑后，至少运行 `npm run build` 验证构建。
- 修改交互或样式后，建议运行 `npm run dev` 并手动试玩几局。
- `src/App.jsx` 目前同时包含业务逻辑和 UI，小改动可以保持现状。
- 如果后续要加入排行榜、账号系统或关卡系统，可以优先把棋盘算法提取为纯函数，方便测试和复用。

## 常见问题

### 为什么开局不会直接消除？

生成棋盘时会检查当前位置左侧两个格子和上方两个格子。如果它们与当前随机水果相同，就会重新随机，直到不会形成开局三连。

### 为什么有些交换不会扣步数？

只有形成消除的交换才是有效交换。无效交换会提示“这一步没有形成三连，换个位置试试”，不会扣除步数。

### 棋盘无解怎么办？

游戏会自动检测是否存在可产生消除的合法交换。如果没有可用交换，会自动刷新棋盘并提示玩家。
